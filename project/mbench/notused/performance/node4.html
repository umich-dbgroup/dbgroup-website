Predicate Selectivity

Selectivity has an impact on both TIMBER and the ORDBMS. The response times of high selectivity queries (QS1 and QS3) are more than the low selectivity queries (QS2 and QS4) by 3 times in the ORDBMS and 8-10 times in TIMBER. In TIMBER, a query with a string predicate (QS1/QS2) performs worse than an equivalent one with an integer predicate (QS3/QS4 resp.) because it accesses a string index which is slower than an integer index. This difference is not observed in the ORDBMS because the lengths of these attributes are short, and the integer index and the short string index are equally effective.
