Pointer-based Join Queries

The difference between these following queries and the join queries based on values (QJ1-QJ2) is that references which can be specified in the DTD or XML Schema and may be optimized with logical OIDs in some XML databases.

QJ3. Low selectivity pointer-based join. Select all OccasionalType nodes that point to a node with aSixtyFour =3. This query returns 1/64 of all the OccasionalType nodes. Selectivity is 1/4096 (0.02%).

QJ4. High selectivity pointer-based join. Select all OccasionalType nodes that point to a node with aFour =3. This query returns 1/4 of all the eOccasional nodes, and the probability of OccasionalType nodes is 1/64. Thus, the selectivity of this query is (0.4%).

Both of these pointer-based joins are semi-join queries. The returned elements are only the eOccasional nodes, not the nodes pointed to.
