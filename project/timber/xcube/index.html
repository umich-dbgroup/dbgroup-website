In Figure 2, +,* in front of the lines indicate differences from the original BUC Algorithm corresponding to the adaptation to support the two properties of XML data mentioned before. In (1*), all duplicate facts are eliminated before the aggregation is computed from the fact attributes. In (4.1+) and (4.2+), we ignore witness trees that have missing dimensions. Ancestors that contain missing dimensions are ignored in (2.1+). Only (1*), which is a support for the non-disjointness property, creates an overhead. We can see that if each grouping tree comprises a large number of individual witness trees, i.e. if the cube is sparse, duplicate elimination can be a bottleneck for this algorithm.
